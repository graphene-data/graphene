---
description: Triage new github issues
---

Use the `gh` command to look at $ARGUMENTS and triage the issue.

If this is a bug, create a test to try and reproduce the issue. Investigate to figure out why the code isn't behaving, and provide an explanation. If there are multiple good ways to fix the problem, lay them out for discussion. If it seems straightforward, go ahead and make the change to fix it.

If this is a feature request, research the code to figure out the possible ways we might implement it, and lay them out for us to discuss. If the feature is small and straightforward, go ahead and implement it and create a test to cover it.
