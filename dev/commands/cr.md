---
description: Kick off a separate code-review process, and address feedback
---

Run `node dev/review.ts` to get feedback on the outstanding change.

One at a time, think through each bit of feedback. Determine if it is valid, and if so make changes to address it.

Add a `## Review` section to task.md. It should have each bit of feedback verbatim, and 1 sentence describing the solution, or why the feedback isn't valid.

After addressing all feedback, take a pass through the code to ensure it is still legible, and clean up if needed.
